# Managed AutoML Milestone 2a 本地 Durable 独立评估

Gemini 独立评估日期：2026-07-23。综合健康度：**85/100**。

## 一、评估总览

- **对象类型**：项目
- **核心结论**：M2a 是目标明确且闭环完整的高质量 PoC，已证明真实上传、持久化、异步人工中断、
  训练、结果下载和重启恢复。它可以在已知小数据、开发者受控操作的前提下用于本地开发或演示。
- **生产判断**：完全不能称为生产可用。安全、可靠性、扩展性、可观测性和运维能力均存在硬门禁。

## 二、核心优势

1. 纵向切片形成了从上传到结果下载的完整、可验证价值流。
2. `lease_generation + control_epoch` fencing 为后续 durable/distributed 演进建立了正确基础。
3. SQLite WAL、job repair 和重启恢复测试证明了限定单机边界下的 durable 能力。

## 三、主要缺陷与风险

1. **无 lease heartbeat**：合法训练超过 30 秒会有错误抢占、重试或活锁风险。
2. **资源管理不足**：CSV/Parquet 全量内存读取且无硬 wall-time 中止，可能造成 OOM 或 worker 卡死。
3. **资源契约不完整**：没有上传大小和内存上限，调用方无法判断安全输入范围。
4. **生产鸿沟**：synthetic auth、SQLite 单点、无正式可观测性/灾备/部署与安全隔离。

## 四、行动建议

- 实现 lease heartbeat，并用超过原租约时间的训练测试证明 job 不被抢占。
- 将训练放入可强制终止的进程边界，实现 `max_wall_time_seconds`，用超时任务验证终止和失败投影。
- 添加明确的数据上传大小限制和资源拒绝响应，并用超限上传验证。
- 后续迁移 PostgreSQL 事务投影，补充指标、告警、OIDC/JWT、RLS 和审计。

最终建议：**补证后推进**。M2a 目标已达成，但应先修复 heartbeat、硬超时和资源上限，再扩展
M2b/M3；不得对外宣称生产可用。
